# Support

Thank you for using BubuStack. Here's how you can get help.

## Self-Service

- **API reference**: The package documentation on pkg.go.dev is the best place to start for exported symbols and examples: https://pkg.go.dev/github.com/bubustack/core
- **Roadmap**: https://bubustack.io/docs/community/roadmap
- **Examples**: See downstream repos and the broader BubuStack GitHub organization for real usage: https://github.com/bubustack

## Community support

For questions, discussions, and community support, please use the following channels:

- **GitHub Issues**: For bug reports and feature requests, please open an issue: https://github.com/bubustack/core/issues
- **GitHub Discussions**: For general questions and cross-repo design discussions, please use Discussions: https://github.com/orgs/bubustack/discussions
- **Discord**: https://discord.gg/dysrB7D8H6

### Triage and response expectations

- We review issues and discussions on a best-effort basis.
- We do not currently provide guaranteed response times or commercial SLAs.
- Security reports follow the process in SECURITY.md.

### Supported versions

- `main` is supported until the first tagged release is published.
- After tagged releases begin, the latest tagged release line is supported.
- Older tagged release lines and untagged local snapshots are unsupported.
- Downstream repos should track the latest supported `core` release line.

## Reporting security vulnerabilities

To report a security vulnerability, please follow the instructions in our [Security Policy](./SECURITY.md). Vulnerabilities must be reported through GitHub Security Advisories.
